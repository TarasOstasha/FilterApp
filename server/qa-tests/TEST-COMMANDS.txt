FILTER-APP — QA test commands reference
======================================

Run all commands from the server/ directory:

  cd server
  npm run <command>


PREREQUISITES
-------------

1. Copy .env.test.example to .env.test and configure your test database.
2. ALLOW_TEST_DATABASE=true is required (safety check in qa-tests/setup.js).
3. DB_NAME should be a dedicated test DB (default rule: name must contain "_test").
4. API tests hit the live Express app via Supertest and need PostgreSQL data loaded
   (products, product_filters, filter_fields) matching your fixture.

Main fixture (flat export from local DB):
  qa-tests/fixtures/product_filters_flat.xlsx

Optional catalog snapshot (not used directly by API tests today):
  qa-tests/fixtures/products.xml

API tests exclude products where hide_product = 'Y' (same as GET /api/products).


GENERAL / LEGACY COMMANDS
-------------------------

npm test
  Runs Mocha with default .mocharc.json (all qa-tests/api + integration tests).

npm run test:watch
  Mocha in watch mode.

npm run test:api
  All API test files under qa-tests/api/**/*.test.js
  Uses default Excel: services/export/data/product_filters.xlsx
  (unless QA_PRODUCTS_EXCEL_PATH is set in .env.test)

npm run test:api:multi
  Only multiFilter.test.js (legacy pivot Excel, not flat profile).

npm run test:integration
  Integration tests only (qa-tests/integration/**/*.test.js).
  Example: product_filters.csv import transaction tests (mock DB).

npm run coverage
  Mocha with NYC code coverage report.


FLAT FIXTURE COMMANDS (recommended)
-----------------------------------

These use qa-tests/runApiTests.js with a profile from qa-tests/config/*.env
and the flat fixture product_filters_flat.xlsx (unless profile says otherwise).

npm run test:api:flat
  Profile: flat-checkbox.env
  File:   productsFromExcel.test.js (checkbox filters only)
  What:   Validates GET /api/products for each checkbox atomic token vs Excel.
          Skips Display_Width, Display_Height, Product_Price.
  Fixture: product_filters_flat.xlsx (~22,856 products)

npm run test:api:multi:flat
  Profile: flat-checkbox.env
  File:   multiFilter.test.js
  What:   ~26 multi-checkbox scenarios (AND combos: Product Type + Backlit,
          Booth Size, etc.). No price, width, or height.

npm run test:api:price:flat
  Profile: flat-price.env
  File:   productPrice.test.js
  What:   Product Price range filters using products.product_price from DB
          (NOT the Excel Product_Price column, which holds filter tiers).
          Tests 5 price buckets + representative point ranges.

npm run test:api:multi-price:flat
  Profile: flat-multi-price.env
  File:   multiFilterPrice.test.js
  What:   Checkbox multi-filter combos + Product Price range together.
          ~24 scenarios; empty combos are skipped.

npm run test:api:dimensions:flat
  Profile: flat-dimensions.env
  File:   dimensionRange.test.js
  What:   Display Width and Display Height range filters only.
          Point ranges (e.g. 48,48) and span ranges (e.g. 48-96).
          20 scenarios (10 per field). Excludes Product_Price only.

npm run test:api:full:flat
  Profile: flat-full.env
  File:   fullFilter.test.js
  What:   Full integration — checkboxes + width + height + price in one query.
          Up to 40 scenario templates; scenarios with zero matching products
          show as "pending" (skipped via this.skip()). Passing = API matches
          Excel filters + DB prices for that combo.

npm run test:api:category:flat
  Profile: flat-category.env
  File:   categoryProducts.test.js
  What:   Validates GET /api/products?catId=X returns every visible product
          whose category_ids in products_check_category.xlsx includes X.
          Tests all unique category_ids (~914) unless capped or filtered.
  Fixture: products_check_category.xlsx (product_code, category_ids)
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:category:flat
  Cap:     QA_MAX_CATEGORIES_TO_TEST=25 npm run test:api:category:flat

npm run test:api:category-filter:flat
  Profile: flat-category-filter.env
  File:   categoryFilter.test.js
  What:   Validates GET /api/products?catId=X&filters… — products must match
          BOTH category_ids in products_check_category.xlsx AND filter columns
          in product_filters_flat.xlsx. Single-checkbox spot checks + multi-filter AND.
  Default: top 5 categories by product count (~50 scenarios max)
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:category-filter:flat

npm run test:api:dynamic-sidebar:flat
  Profile: flat-dynamic-sidebar.env
  File:   dynamicSidebar.test.js
  What:   Validates sidebar checkbox facet options:
          GET /api/filterField?catId=X (static initial sidebar)
          GET /api/dynamic-filters?catId=X&filters… (after user selects filters)
          Scenarios from Excel fixtures; expected facets computed with the same
          SQL scope as the API (category join + active filter joins).
  Default: top 5 categories static + up to 50 dynamic scenarios
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:dynamic-sidebar:flat

npm run test:api:range-endpoints:flat
  Profile: flat-range-endpoints.env
  File:   rangeEndpoints.test.js
  What:   Validates slider rail endpoints vs reference SQL:
          GET /api/products/price-range (min, max, breakpoints)
          GET /api/products/width-range (min, max, globalMin, globalMax)
          GET /api/products/height-range (min, max, globalMin, globalMax)
          Scenarios: global, catId only, catId + checkbox filters.
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:range-endpoints:flat

npm run test:api:pagination-sort:flat
  Profile: flat-pagination-sort.env
  File:   paginationSort.test.js
  What:   Validates GET /api/products pagination and sort:
          totalProducts stable across pages; limit/offset slices match reference;
          no duplicate product_codes across pages; sortBy most_popular | price_asc | price_desc.
  Default: scenarios with ≥30 products; unions verified up to 600 (configurable).
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:pagination-sort:flat

npm run test:api:mega:flat
  Profile: flat-mega.env
  File:   megaFilter.test.js
  What:   Validates GET /api/products/mega (Mega Filter search bar):
          searchTerms on product_name / product_code (word AND), optional catId,
          sortBy most_popular | price_asc | price_desc, max 54 results.
          Code/prefix scenarios from Excel + multi-word searches from DB catalog.
  Focused: QA_CATEGORY_IDS_TO_TEST=161 npm run test:api:mega:flat


DIAGNOSTIC / DATA TOOLS (not Mocha tests)
-----------------------------------------

npm run test:compare-excel-db
  Script: qa-tests/scripts/compareExcelDb.js
  What:   Compares Excel atomic checkbox tokens vs product_filters in DB.
          Reports product_code mismatches, token diffs, and count mismatches.
          Does NOT use hide_product or test the API. Always exits 0 (report only).

npm run export:db-only-products
  Script: qa-tests/scripts/exportDbOnlyProducts.js
  What:   Exports product_codes that exist in DB but not in the Excel fixture.
          Output: qa-tests/output/db-only-product-codes.csv


PROFILE CONFIG FILES (qa-tests/config/)
---------------------------------------

flat-checkbox.env
  QA_PRODUCTS_EXCEL_PATH=qa-tests/fixtures/product_filters_flat.xlsx
  QA_EXCLUDE_TEST_FIELDS=Display_Width,Display_Height,Product_Price

flat-price.env
  QA_TEST_TIMEOUT_MS=180000

flat-multi-price.env
  Flat Excel + exclude width/height/price from productsFromExcel (price from DB)

flat-dimensions.env
  Flat Excel + QA_EXCLUDE_TEST_FIELDS=Product_Price only

flat-full.env
  Flat Excel, all filter types enabled, 300s timeout

flat-category.env
  QA_CATEGORY_EXCEL_PATH=qa-tests/fixtures/products_check_category.xlsx
  600s timeout (full category sweep can take several minutes)

flat-category-filter.env
  Flat filter Excel + category Excel, checkbox multi-filter combos with catId

flat-dynamic-sidebar.env
  Static filterField + dynamic-filters facet validation vs Excel

flat-range-endpoints.env
  price-range, width-range, height-range rail endpoints

flat-pagination-sort.env
  GET /api/products limit, offset, sortBy validation

flat-mega.env
  GET /api/products/mega search + catId + sortBy


OPTIONAL .env.test VARIABLES
----------------------------

QA_PRODUCTS_EXCEL_PATH     Path to Excel fixture (overrides default pivot file)
QA_PRODUCTS_EXCEL_SHEET    Sheet name if workbook has multiple sheets
QA_EXCLUDE_TEST_FIELDS     Comma-separated fields to skip in productsFromExcel tests
                           e.g. Display_Width,Display_Height,Product_Price
QA_MAX_UNIQUE_VALUES_PER_FILTER   Cap checkbox values tested per field (default: all)
QA_TEST_TIMEOUT_MS         Per-test timeout in milliseconds
QA_DB_ONLY_EXPORT_PATH     Custom path for export:db-only-products output
QA_CATEGORY_EXCEL_PATH     Path to category fixture (product_code + category_ids)
QA_CATEGORY_IDS_TO_TEST    Comma-separated category_ids to test (e.g. 161,169)
QA_MAX_CATEGORIES_TO_TEST  Cap how many categories are tested (ordered by id)
QA_MAX_CATEGORY_FILTER_SCENARIOS  Cap category+filter scenarios (default 50)
QA_MAX_RANGE_ENDPOINT_SCENARIOS   Cap range endpoint scenarios (default 35)
QA_MAX_PAGINATION_SCENARIOS       Cap pagination test scenarios (default 12)
QA_PAGINATION_PAGE_SIZE           Page size for pagination tests (default 27)
QA_MAX_PAGINATION_FULL_UNION      Max products to fully paginate per scenario (default 600)
QA_MIN_PRODUCTS_FOR_PAGINATION    Min products required for a scenario (default 30)
QA_MAX_MEGA_SCENARIOS               Cap mega code/prefix scenarios (default 10)
QA_MAX_MEGA_MULTI_WORD              Cap multi-word mega scenarios (default 3)


TEST FILES MAP (qa-tests/api/)

MOCHA RESULT LEGEND
-------------------

passing   Test ran and all assertions passed.
failing   Test ran and an assertion failed (investigate API vs Excel/DB drift).
pending   Test was skipped (this.skip()) — usually "no products match this filter
          combo" in fullFilter tests. Not a failure; exit code is still 0.

7. npm run test:api:full:flat        — full integration
8. npm run test:api:category:flat    — category view (catId)
9. npm run test:api:category-filter:flat — category + checkbox filters
10. npm run test:api:dynamic-sidebar:flat — sidebar facet options (static + dynamic)
11. npm run test:api:range-endpoints:flat — price / width / height range rails
12. npm run test:api:pagination-sort:flat — pagination + sorting
13. npm run test:api:mega:flat — Mega Filter search
